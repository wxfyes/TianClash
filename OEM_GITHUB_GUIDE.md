# 🚀 GitHub 在线一键打包教程 (OEM)

本教程将指导你如何使用 GitHub Actions 在线为不同客户打包定制化的客户端。你无需在本地安装任何开发环境，只需在网页上操作即可。

## ✨ 功能特点

- **全自动修改**: 自动替换应用名称、包名、图标、接口地址。
- **多平台支持**: 一次运行，同时生成 Android APK 和 Windows 客户端。
- **云端构建**: 利用 GitHub 服务器编译，不占用本地电脑资源。

## 📋 准备工作

在开始之前，请准备好以下客户信息：
1.  **应用名称**: 客户想要显示在桌面的名字 (例如: "极速VPN")。
2.  **包名 (Package Name)**: Android 唯一标识符 (例如: "com.speed.vpn")。
    *   *注意: 包名必须是唯一的，通常使用 `com.公司名.应用名` 的格式。*
3.  **OSS 接口地址**: 客户提供的配置接口 URL (例如: `https://oss.example.com/config.txt`)。
4.  **应用图标 URL**: 一张高清图标的直链地址 (推荐 1024x1024 PNG)。
    *   *提示: 你可以把图标上传到图床，然后获取直接链接。*

## 🛠️ 操作步骤

### 第一步：进入 Actions 页面

1.  打开项目的 GitHub 仓库页面。
2.  点击顶部导航栏的 **Actions** 标签。
3.  在左侧列表中，点击 **Build OEM Client**。

### 第二步：填写参数

1.  点击右侧的灰色按钮 **Run workflow**。
2.  在弹出的表单中填写客户信息：

    | 参数名 | 说明 | 示例 |
    | :--- | :--- | :--- |
    | **App Name** | 应用名称 | `极速VPN` |
    | **Package Name** | 包名 (Android ID) | `com.speed.vpn` |
    | **OSS URL** | 配置接口地址 | `https://oss.xx.com/conf` |
    | **ImgBB API Key** | 图片上传Key (一般不用改) | `85719...` |
    | **Icon URL** | **图标直链 (重要)** | `https://example.com/logo.png` |

    > **关于图标**: 只需要提供一个 URL，系统会自动裁剪生成 Android 和 Windows 所需的所有尺寸图标。

3.  确认信息无误后，点击绿色的 **Run workflow** 按钮。

### 第三步：等待编译

1.  点击按钮后，页面上会出现一个新的运行记录 (黄色圆圈表示正在运行)。
2.  整个编译过程大约需要 **10-15 分钟**。
3.  你可以点击进去查看详细的编译日志。

### 第四步：下载安装包

1.  当运行状态变成 **绿色对勾 (✅)** 时，表示打包成功。
2.  点击该条运行记录进入详情页。
3.  向下滚动到页面底部的 **Artifacts** 区域。
4.  点击下载你需要的文件：
    *   📥 **xxx-Android**: 包含 Android APK 安装包。
    *   📥 **xxx-Windows**: 包含 Windows 绿色版压缩包。

## ❓ 常见问题

**Q: 图标 URL 填错了怎么办？**
A: 如果 URL 无法访问，系统会自动使用默认的"天阙"图标，不会导致打包失败。

**Q: 包名有什么限制吗？**
A: 包名只能包含字母、数字和点(.)，且不能以数字开头。例如 `com.app.123` 是合法的，但 `123.com.app` 是非法的。

**Q: 为什么 Windows 包下载下来是 ZIP？**
A: Windows 编译生成的是绿色版文件夹，我们将其压缩为 ZIP 方便下载。解压后直接运行 `.exe` 即可。

**Q: 编译失败了怎么办？**
A: 点击失败的运行记录，查看 "Build" 步骤的日志。常见的失败原因包括：网络问题、参数填写包含非法字符等。如果是网络问题，重新运行一次通常就能解决。
